# Workflow: Tennis Edge Dashboard

## Objective
Run the Streamlit dashboard that combines tournament schedules, Kalshi prediction markets,
and live ATP/WTA model predictions with edge calculations.

## Required Inputs
- `tennis_model/player_elos.json` (ATP) — generated by `tennis_model/backtest.py`
- `tennis_model/wta/player_elos.json` (WTA) — generated by `tennis_model/wta/backtest.py`
- Internet connection (model fetches recent Sackmann match data on startup)

## How to Run

```bash
# Install dependencies (first time only)
pip install -r requirements.txt

# Launch the dashboard
streamlit run tennis_dashboard.py
```

Opens at http://localhost:8501

## Tabs

### Tab 1 — Schedule
- Shows upcoming ATP/WTA 1000 + Grand Slams for the next N days
- Filter by tour (ATP / WTA / both) and lookahead window
- Green highlight = tournament currently in progress
- Data source: hardcoded 2026 calendar in `tools/tennis_schedule.py`

**When dates shift**: edit `tools/tennis_schedule.py` and update the relevant `start_date`/`end_date`
fields. No other changes needed.

### Tab 2 — Kalshi Markets
- Fetches open tennis prediction markets from Kalshi's public API
- Search by player name or tournament
- Click any row to see bid/ask, volume, and implied probability
- Click "Use this market in Edge Analysis" to prefill Tab 3 with the market odds

**No API key yet**: works unauthenticated (read-only market data). Add `KALSHI_API_KEY=your_key`
to `.env` to unlock additional data.

### Tab 3 — Edge Analysis
- Select ATP or WTA, two players, surface, best-of format
- Optionally paste in market implied probability (or link from Tab 2)
- Model computes win probability using the composite Elo + stats model
- Edge = Model Win% − Market Implied Win%
- Threshold: ±4% = VALUE signal (green = model favors P1, red = model favors P2)
- Component bar chart shows what's driving the prediction

## Data Freshness
- Elo ratings: updated by `backtest.py` / `wta/backtest.py`. Run those when ratings feel stale.
- Rolling stats (serve%, form, etc.): fetched live from Sackmann GitHub on each dashboard startup
  (cached for 1 hour within a session)
- Kalshi markets: fetched live; hit "Refresh" in Tab 2 to re-pull

## Troubleshooting

| Issue | Fix |
|---|---|
| `player_elos.json not found` | Run `python tennis_model/backtest.py` first |
| Kalshi tab shows error | Check internet; or add API key to `.env` |
| Players not found in selectbox | Run `backtest.py` to refresh Elo file; or type partial name to filter |
| Model loads slowly on first open | Normal — fetching 2 years of Sackmann data (~30s). Cached after that. |
